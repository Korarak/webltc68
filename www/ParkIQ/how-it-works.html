System Overview

ทำงานอย่างไร? รู้จัก ParkIQ ตั้งแต่เซ็นเซอร์ถึงแดชบอร์ด

ParkIQ ใช้สถาปัตยกรรมที่ผสานฮาร์ดแวร์ IoT, AI Vision และแพลตฟอร์มเว็บแบบเรียลไทม์ เพื่อให้ข้อมูลที่จอดรถไหลลื่นจากพื้นที่จริงสู่ทีมบริหารในทันที

Hardware Layer

ESP32 + Infrared + LED + ESP32-CAM

Data Layer

MQTT → API Gateway → Real-time Engine

Experience Layer

Dashboard + Booking + Automation

Hardware Flow

ฮาร์ดแวร์ตรวจจับ 2 ชั้นเพื่อความแม่นยำสูงสุด

เซ็นเซอร์อินฟราเรดตรวจสอบสถานะเบื้องต้น ก่อนส่งต่อให้ ESP32-CAM ยืนยันด้วยการประมวลผลภาพ ทุกขั้นตอนใช้โปรโตคอลที่ออกแบบมาเพื่อความรวดเร็วและเสถียร

1

Infrared Detection

ตรวจจับวัตถุที่ความละเอียดสูง พร้อมตั้งค่า Sensitivity แยกแต่ละช่องจอด

2

AI Vision Validation

ESP32-CAM ส่งภาพไปยังโมเดล AI เพื่อตรวจสอบสถานะรถพร้อม Confidence Score

3

Feedback Loop

ผลลัพธ์ถูกส่งกลับไปควบคุม LED แสดงสถานะและบันทึกลงฐานข้อมูลพร้อม Timestamp

Key Hardware Specs

  • • ESP32 Dual Board รองรับ 16 ช่องจอดต่อชุด
  • • ESP32-CAM พร้อมเลนส์กว้างและไฟเติมแสง
  • • อินฟราเรดเซ็นเซอร์พร้อมฝาครอบป้องกันแสง
  • • LED แสดงสถานะความสว่างสูง เห็นชัดทั้งกลางวันและกลางคืน

Communication

  • • ใช้ MQTT สำหรับการส่งข้อมูลสถานะเบื้องต้น
  • • HTTP/HTTPS สำหรับการอัปเดตข้อมูลสำคัญและคำสั่งควบคุม
  • • WebSocket สำหรับการแจ้งเตือนเหตุการณ์แบบทันที
Software Pipeline

ข้อมูลเดินทางอย่างไรภายใน ParkIQ

เมื่อข้อมูลออกจากฮาร์ดแวร์ ทุกแพ็กเก็ตจะผ่านการประมวลผลอย่างเป็นระบบ เพื่อให้ผู้ใช้งานได้รับข้อมูลที่ถูกต้องและรวดเร็วเสมอ

1

Edge Processing

บอร์ด ESP32 ทำการกรองข้อมูล Noise และแนบ Timestamp ก่อนส่ง

  • • ใช้ Debounce Algorithm เพื่อลดการส่งข้อมูลซ้ำซ้อน
  • • ตรวจสอบความถูกต้องของข้อมูลจากเซ็นเซอร์ก่อนส่ง
  • • ประมวลผลเบื้องต้นเพื่อลดภาระของเซิร์ฟเวอร์
2

API Gateway

ข้อมูลเข้าสู่ API Gateway ที่ตรวจสอบ Token และกำหนด Rate Limit

  • • ตรวจสอบ Authentication และ Authorization
  • • จำกัดจำนวน Request ต่อวินาทีเพื่อป้องกัน Overload
  • • Logging และ Monitoring ทุก Request
3

Real-time Engine

อัปเดตห้อง WebSocket เพื่อกระจายข้อมูลไปยัง UI และระบบแจ้งเตือน

  • • ใช้ WebSocket สำหรับการส่งข้อมูลแบบ Real-time
  • • รองรับการเชื่อมต่อพร้อมกันหลาย Client
  • • Auto-reconnect เมื่อการเชื่อมต่อขาด
4

Data Lake & Analytics

บันทึกลงฐานข้อมูลสำหรับการสร้างรายงานและการวิเคราะห์ย้อนหลัง

  • • เก็บข้อมูลแบบ Time-series สำหรับการวิเคราะห์
  • • รองรับการ Query ข้อมูลย้อนหลังได้อย่างรวดเร็ว
  • • สร้างรายงานอัตโนมัติตามช่วงเวลาที่กำหนด

UI Update Strategy

  • • ใช้ Zustand/Signal จัดการสถานะในฝั่ง Frontend
  • • Partial Updates เพื่อลดการ Render ทั้งหน้า
  • • แสดง Skeleton Loading ระหว่างรอข้อมูลสำรอง

Alert Engine

  • • ตั้ง Threshold สำหรับอัตราการใช้พื้นที่
  • • แจ้งเตือนรถจอดเกินเวลาที่กำหนด
  • • รองรับการส่ง Webhook ไปยังระบบอื่น
Automation Features

ระบบอัตโนมัติที่ทำงานให้คุณตลอดเวลา

ParkIQ ไม่ได้หยุดแค่แสดงผล แต่ดำเนินการแทนคุณในหลายสถานการณ์ เพื่อช่วยลดงานซ้ำและเพิ่มความพึงพอใจของผู้ใช้งาน

Smart Booking

  • • ตรวจสอบช่องว่างก่อนยืนยันจอง
  • • กันพื้นที่อัตโนมัติเมื่อจองสำเร็จ
  • • ส่ง QR Payment พร้อมเวลาหมดอายุ

Operational Alerts

  • • แจ้งเตือนช่องว่างเหลือน้อย
  • • สร้างทริกเกอร์เปิดปิดแถบกั้น
  • • ส่งรายงานรายวันผ่านอีเมล

Insight & Forecast

  • • วิเคราะห์ช่วงเวลาพีคอัตโนมัติ
  • • คาดการณ์ความจุในอนาคตอันใกล้
  • • แนะนำการจัดสรรโซนพิเศษชั่วคราว
Security & Reliability

ความปลอดภัยและเสถียรภาพระดับองค์กร

ParkIQ ถูกออกแบบให้รองรับการใช้งานระยะยาว ทั้งด้านความปลอดภัยของข้อมูลและความมั่นคงของระบบ

  • TLS Encryption ครอบคลุมทุกการสื่อสารระหว่างอุปกรณ์และเซิร์ฟเวอร์
  • Role Based Access Control แยกสิทธิ์ผู้ใช้ตามบทบาท
  • Audit Log เก็บประวัติการทำงานทุกคำสั่งเพื่อการตรวจสอบย้อนหลัง
  • Fallback และ Retry Mechanism เมื่อการเชื่อมต่อขัดข้อง

High Availability

ออกแบบให้รองรับการทำงานแบบ Fault Tolerant พร้อมระบบ Health Check ที่ตรวจสอบสถานะแต่ละบริการ

Data Protection

สำรองข้อมูลอัตโนมัติ ร่วมกับการเข้ารหัสข้อมูลสำคัญทั้งขณะส่งและขณะพัก

Scalability

รองรับการเพิ่มจำนวนช่องจอดและสาขาได้ทันที ผ่านการเพิ่ม Node ของ ESP32 และการขยายโครงสร้าง Backend
